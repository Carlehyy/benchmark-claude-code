# 性能优化

## 模型选择策略

**Haiku 4.5**（Sonnet 能力的 90%，成本节省 3 倍）：
- 轻量级代理，频繁调用
- 配对编程与代码生成
- 多代理系统中的工作代理

**Sonnet 4.5**（最佳编码模型）：
- 主要开发工作
- 多代理工作流的协调
- 复杂编码任务

**Opus 4.5**（最深层推理）：
- 复杂架构决策
- 最大推理需求
- 研究与分析任务

## 上下文窗口管理

避免在上下文窗口的最后 20% 使用于：
- 大规模重构
- 跨多个文件的功能实现
- 调试复杂交互

上下文敏感度较低的任务：
- 单文件编辑
- 独立工具创建
- 文档更新
- 简单错误修复

## Ultrathink + 计划模式

针对需要深度推理的复杂任务：
1. 使用 `ultrathink` 进行增强思考
2. 启用**计划模式**以实现结构化方法
3. 通过多轮批判“加速引擎”
4. 使用分角色子代理进行多样化分析

## 构建故障排除

如果构建失败：
1. 使用**build-error-resolver**代理
2. 分析错误信息
3. 逐步修复
4. 每次修复后进行验证
