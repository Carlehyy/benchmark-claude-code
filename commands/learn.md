# /learn - 提取可复用模式

分析当前会话，提取值得保存为技能的模式。

## 触发条件

在会话中任何解决了非平凡问题的时刻运行 `/learn`。

## 提取内容

关注以下内容：

1. **错误解决模式**
   - 发生了什么错误？
   - 根本原因是什么？
   - 如何修复的？
   - 是否可复用于类似错误？

2. **调试技巧**
   - 非显而易见的调试步骤
   - 有效的工具组合
   - 诊断模式

3. **变通方法**
   - 库的特殊情况
   - API 限制
   - 版本特定的修复

4. **项目特定模式**
   - 发现的代码库约定
   - 做出的架构决策
   - 集成模式

## 输出格式

在 `~/.claude/skills/learned/[pattern-name].md` 创建技能文件：

```markdown
# [描述性模式名称]

**Extracted:** [日期]
**Context:** [适用场景简述]

## Problem
[解决的问题 - 具体说明]

## Solution
[模式/技巧/变通方法]

## Example
[如适用，代码示例]

## When to Use
[触发条件 - 何时激活此技能]
```

## 处理流程

1. 审查会话，寻找可提取的模式
2. 确定最有价值/可复用的见解
3. 草拟技能文件
4. 询问用户确认后保存
5. 保存至 `~/.claude/skills/learned/`

## 注意事项

- 不提取琐碎修复（拼写错误、简单语法错误）
- 不提取一次性问题（特定 API 故障等）
- 关注能节省未来会话时间的模式
- 保持技能聚焦 - 每个技能只包含一个模式
