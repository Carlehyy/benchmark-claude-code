# 测试要求

## 最低测试覆盖率：80%

测试类型（全部必需）：
1. **单元测试** - 单个函数、工具函数、组件
2. **集成测试** - API 端点、数据库操作
3. **端到端测试（E2E）** - 关键用户流程（Playwright）

## 测试驱动开发

强制执行的工作流程：
1. 先编写测试（红灯阶段）
2. 运行测试 - 应该失败
3. 编写最小实现代码（绿灯阶段）
4. 运行测试 - 应该通过
5. 重构代码（改进阶段）
6. 验证覆盖率（80%以上）

## 测试失败排查

1. 使用 **tdd-guide** 代理
2. 检查测试隔离性
3. 验证模拟对象是否正确
4. 修正实现代码，而非测试代码（除非测试本身有误）

## 代理支持

- **tdd-guide** - 主动用于新功能开发，强制先写测试
- **e2e-runner** - Playwright 端到端测试专家
